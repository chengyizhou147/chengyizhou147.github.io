提权

查找sudo权限命令，需要密码。
sudo -l

SUID权限可执行文件，发现有exim4，查看具体版本后发现是4.92。存在本地提权漏洞(local root exploit)的版 本为exim <= 4.84-3


常见的SUID提权可执行文件
nmap
vim
less
more
nano
cp
mv
find
wget
bash
/var/log/ona.log为OpenNetAdmin的日志文件，查看之后发现无可利用内容。
/var/mail/www-data为www-data用户的邮件，查看之后也无可利用内容。
然后想到还有个reports目录，然后发现该目录下有一个.htaccess文件

该文件又将我们引向/var/www/.htpasswd文件
访问该文件

这个应该是douglas的密码hash值，下面应该是提示，
＃为了减轻痛苦（标准词典可能会失败），
＃为此10个字符的密码使用以下字符集：aefhrt
意思是用这6个字母组成10个字母的密码。我们可以使用crunch生成爆破字典

然后再去爆破hash

得出密码为fatherrrrr
使用ssh登录


使用sudo -l命令查看当前用户有哪些root权限，看到cp(复制命令被赋予root)

这应该是要让我们登录到jen这个账户里面去，具体方法：

这个时候我们来到 /tmp（临时目录）下给 authorized_keys 这个文件一个可执行权限，然后使用 sudo以 jen 用户权限去执行 cp 命令，把我们刚刚生成的 authorized_keys移动到 /home/jen/.ssh/ 目录下，这样我们才能登陆它
